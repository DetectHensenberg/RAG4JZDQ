# ALIGNMENT — 需求3：标书商务文件编写助手

## 项目上下文

- **技术栈**：FastAPI + Vue3 + ElementPlus + TailwindCSS + ChromaDB + BM25 + DashScope LLM
- **前端设计**：暗黑毛玻璃风格（backdrop-blur, rgba白边框, #0a0a0a背景, 渐变边框, 16-20px圆角）
- **现有页面**：知识库问答、知识库构建、系统总览、数据浏览、摄取管理、评估面板、系统配置、标书助手（需求1+需求2）
- **标书助手**：已有"产品真伪辨别"和"业绩库检索"两个功能Tab

## 原始需求摘要

707部每年有大量标书编写工作，商务文件内容高度重复（企业资质、财务报表、声明函、证照等）。需要：
1. 自动解析招标文件，识别商务文件条款要求
2. 根据既定模板生成商务文件大纲
3. 调用投标知识库填充重复性内容
4. 为证照文件（身份证、营业执照、资质证书）自动添加水印
5. 根据招标文件模板自动填充承诺书等内容

## 已确认决策

| # | 问题 | 决策 |
|---|------|------|
| P0-1 | 投标知识库数据来源 | **新建独立 collection `bid_materials`**，专门存储公司资质、模板、证照等 |
| P0-2 | 证照文件存储方式 | **文件系统 + 元数据 SQLite**，类似需求2的附件管理 |
| P0-3 | 水印添加方式 | **后端 pdf2image + PIL 处理**，支持PDF和图片 |
| P1-1 | 交互方式 | **混合模式**：向导式上传招标文件 → 对话式生成内容 |
| P1-2 | 大纲生成后如何编辑 | **结构化表格**，可拖拽排序、勾选章节 |
| P1-3 | 最终输出格式 | **多格式支持**，优先 Word(.docx) |
| P2-1 | 承诺书模板管理 | **后台可配置**，支持上传模板 |

## 需求边界（待确认后更新）

### 初步在范围内

1. 标书助手新增"商务文件编写"Tab（第三个功能）
2. 招标文件上传与解析（PDF/DOCX）
3. 商务文件条款识别（LLM提取）
4. 大纲生成与编辑（基于模板）
5. 投标知识库管理（公司资质、证照、模板）
6. 水印添加功能（项目编号水印）
7. 内容填充与导出（Word格式）

### 初步不在范围内

- 需求4（后续迭代）
- 复杂排版（保持简洁Word输出）
- OCR识别（假设上传的是可解析PDF）

## 数据模型（草案）

### 投标材料表 `bid_materials`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| name | TEXT NOT NULL | 材料名称（如"营业执照"） |
| category | TEXT | 分类（资质证书/财务报表/声明函/证照/模板） |
| file_path | TEXT | 文件路径 |
| content | TEXT | 文本内容（用于检索） |
| valid_from | TEXT | 有效期起始 |
| valid_to | TEXT | 有效期截止 |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

### 商务文件模板表 `bid_templates`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| name | TEXT NOT NULL | 模板名称（如"承诺书模板"） |
| template_type | TEXT | 类型（承诺书/声明函/公司介绍） |
| content | TEXT | 模板内容（支持变量占位符） |
| created_at | TEXT | 创建时间 |

## 工作流程（草案）

```
Step 1: 上传招标文件（PDF/DOCX）
        ↓
Step 2: LLM解析商务文件条款要求
        ↓
Step 3: 生成商务文件大纲（可编辑）
        ↓
Step 4: 用户确认/调整大纲
        ↓
Step 5: 系统从知识库检索并填充内容
        ↓
Step 6: 用户输入项目编号，系统为证照添加水印
        ↓
Step 7: 导出Word文档
```
