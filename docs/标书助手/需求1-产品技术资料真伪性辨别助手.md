# 需求1：产品技术资料真伪性辨别助手

> 优先级：**高**

---

## 一、背景

707部在开展系统集成项目投标工作时，会涉及到大量通用硬件设备的产品选型，如：服务器、交换机、摄像头、音响等。需要根据招标技术要求，联系产品厂家提供产品彩页、第三方检测报告、技术白皮书等技术资料，并将技术资料整理后补充到投标文件中，作为应标的技术支撑材料。

但个别厂家为了应标，在自身产品不满足招标技术要求的情况下，隐瞒投标方，私自将产品彩页、检测报告、技术白皮书等资料中的技术参数篡改为投标文件要求的指标参数，待评标时被专家或竞争对手发现将导致**废标及虚假应标的风险**，严重情况下会将公司拉入"黑名单"，禁止参加投标，后果非常严重。

因此，当前707部的售前技术人员会花大量时间对招标文件技术要求、产品厂家技术参数、厂家官方公开产品参数以及之前厂家提供的同型号产品技术参数进行**人工比对**，消耗大量人力和时间，还存在一定的漏项、错项等情况。

**急需采用先进、可靠的技术手段来解决以上问题，坚决杜绝"资料作假"、"虚假应标"等风险。**

---

## 二、具体需求

通过大模型、智能体应用等技术手段，论证并实现：

1. **产品匹配**：根据招标文件的技术要求，快速从产品知识库里查找匹配到本次投标选用的产品型号和厂家
2. **资料上传**：将厂家提供的产品彩页、检测报告、技术白皮书等资料上传到智能体应用平台
3. **参数比对**：由大模型去比对分析厂家技术资料中的技术参数真实性，如果有偏差，进行提醒并给出偏差内容及资料位置
4. **辅助选型**：通过上传招标文件技术要求后，平台自动分析并推荐出满足招标要求的相关产品厂家和产品型号，提升产品选型的科学性、规范性

### 核心价值

- 将标书编写人员从海量的参数比对工作中解脱释放出来
- 更有精力去提高标书整体质量
- 杜绝虚假应标风险
- 提高投标工作的安全和可控性

---

## 三、场景举例

1. 707部获取到A客户的招标文件，其中明确摄像头的分辨率为1080P
2. 标书编写人员将该摄像头名称及分辨率等技术参数通过对话框的形式输入给智能体应用平台-产品技术资料真伪性辨别助手，智能体应用平台就会到产品知识库中搜索匹配摄像头参数的厂家、品牌、型号、具体技术参数、厂家联系方式等反馈标书编写人员
3. 标书编写人员通过综合比较多个厂家的产品，确定投标产品型号，收集厂家提供的相关产品彩页、检测报告、技术白皮书等支撑材料
4. 标书编写人员将厂家提供的技术支撑材料导入到智能体应用平台与产品知识库中的同型号产品技术指标参数进行自动比对，并将比对结果在界面上进行提醒和展示

---

## 四、功能拆解

| 功能模块 | 说明 |
|----------|------|
| 产品知识库管理 | 维护产品厂家、型号、技术参数等基础数据 |
| 招标需求解析 | 从招标文件中提取技术要求和参数指标 |
| 智能产品匹配 | 根据技术要求自动推荐匹配的产品 |
| 资料上传与解析 | 上传厂家技术资料并自动提取参数 |
| 参数真伪比对 | 比对厂家资料参数与知识库/官方参数的差异 |
| 偏差报告生成 | 展示比对结果，标注偏差内容和资料位置 |
