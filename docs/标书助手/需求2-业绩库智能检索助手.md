# 需求2：业绩库智能检索助手

> 优先级：**中**

---

## 一、背景

707部在进行集成项目投标时，会根据招标要求提供类似项目的业绩证明材料，包括项目名称、项目内容、项目金额、合同签订时间、验收时间、甲方联系人及联系方式等。

707部近几年来履约了**数百个集成项目**，项目类型繁多，查找十分不便，当标书编写时都需要去挨个查找之前的合同和案例，消耗大量的人力和时间在做重复的事情。

---

## 二、具体需求

### 2.1 业绩数据管理

通过智能体应用平台，将公司历史履约的合同（非密）基本信息读取并**列表展示**出来，并提供可编辑和管理的功能：

| 字段 | 说明 |
|------|------|
| 项目名称 | 合同项目名称 |
| 项目内容 | 项目描述和范围 |
| 项目金额 | 合同金额 |
| 合同签订时间 | 签约日期 |
| 验收时间 | 项目验收日期 |
| 甲方联系人 | 客户方联系人 |
| 联系方式 | 客户方联系方式 |

### 2.2 智能检索

标书编写人员在编写业绩证明材料时，通过**问答方式**能快速搜索到：

- 历史履约同类项目的基本信息
- 合同原始文件
- 入账凭证
- 验收材料

可以选择将相关项目的业绩材料**下载并插入**到标书里，有效提升业绩查询的效率和准确率。

### 2.3 业绩库闭环管理

平台应支持快速、便捷地增加业绩合同内容，丰富完善业绩库，提供业绩库的**闭环管理能力**。

---

## 三、场景举例

1. 707部获取到了A客户的智慧城市集成项目的招标文件，其中明确要求要提供**近三年合同金额在500万以上**的类似项目业绩证明材料，需要附上合同原始文件
2. 标书编写人员利用业绩库智能检索助手，通过输入关键字"智慧城市"、筛选履约时间为2022年1月至2024年12月，合同金额≥500万，检索到历史业绩库中符合要求的合同摘要信息和合同原件
3. 标书人员按招标文件要求将摘要信息改写后就可以复制到投标文件中，并插入合同原件

---

## 四、功能拆解

| 功能模块 | 说明 |
|----------|------|
| 业绩数据录入 | 支持手动录入和批量导入业绩合同信息 |
| 业绩列表展示 | 列表展示所有业绩记录，支持排序和分页 |
| 多条件筛选 | 按关键字、时间范围、金额范围等条件筛选 |
| 智能问答检索 | 通过自然语言问答方式检索业绩 |
| 附件管理 | 关联合同原件、入账凭证、验收材料等附件 |
| 材料下载与导出 | 支持下载业绩材料，插入到标书中 |
| 业绩编辑与维护 | 支持新增、修改、删除业绩记录 |
