# ALIGNMENT — 需求1：产品技术资料真伪性辨别助手

## 项目上下文

- **技术栈**：FastAPI + Vue3 + ElementPlus + TailwindCSS + ChromaDB + BM25 + DashScope LLM
- **前端设计**：暗黑毛玻璃风格（backdrop-blur, rgba白边框, #0a0a0a背景, 渐变边框, 16-20px圆角）
- **现有页面**：知识库问答、知识库构建、系统总览、数据浏览、摄取管理、评估面板、系统配置
- **侧边栏**：路由驱动，左侧导航栏，每个路由一个入口
- **现有问答**：ChatView.vue — 对话式交互，SSE 流式输出，参考来源展示

## 原始需求

707部在投标过程中需要比对厂家送审资料与官方技术参数，防止厂家篡改参数导致废标。
需要：产品匹配 → 资料上传 → 参数比对 → 偏差报告，以此顺序操作。

## 关键决策（用户确认）

| # | 问题 | 用户回答 |
|---|------|----------|
| P0-1 | 产品知识库数据来源 | **暂时为空**，先做功能框架，后续手动添加 |
| P0-2 | 对比基准 | 知识库 = 可信基准（官方白皮书），送审 = 待验证 |
| P0-3 | 操作流程 | **顺序工作流**（Step1→Step2→Step3），不是独立入口 |
| P1-1 | 前端交互形式 | **对话式**（类似现有知识库问答 ChatView） |
| P1-2 | 偏差报告展示 | **两者都要**（Markdown 流式 + 结构化表格） |
| P1-3 | 资料位置定义 | **PDF 页码 + 章节** |

## 需求边界

### 在范围内

1. 左侧导航栏新增"标书助手"入口（与"知识库问答"同级）
2. 标书助手页面 — **对话式交互**，类似 ChatView，但引导顺序工作流
3. 顺序工作流：招标需求输入 → 产品匹配推荐 → 上传送审资料 → 参数比对报告
4. 产品知识库（独立 collection `products`）— 框架先行，数据后补
5. 参数比对报告：Markdown 流式输出 + 结构化可排序表格
6. 偏差定位：PDF 页码 + 章节标注

### 不在范围内

- 需求2/3/4（后续迭代）
- 证照水印功能
- 业绩库管理
- 产品知识库的数据填充（用户自行添加）

## 需求理解

### 用户操作场景（顺序）

```
用户: 输入招标技术要求（如"摄像头 分辨率1080P IP67"）
系统: 从产品知识库检索匹配产品，展示推荐列表（厂家、型号、参数、联系方式）
用户: 选定产品，上传厂家送审的技术资料（PDF/DOCX）
系统: 自动提取送审资料中的技术参数
系统: 与知识库中该产品的官方参数比对
系统: 流式输出偏差报告（Markdown表格），并标注偏差在PDF中的页码和章节
同时: 在前端渲染结构化表格（可排序、可筛选）
```

### 与现有系统的关系

- 复用现有 `HybridSearch` 检索能力（指向 `products` collection）
- 复用现有 `IngestionPipeline` 摄取产品文档
- 复用现有 `LLMFactory` 调用大模型
- 复用现有 SSE 流式输出模式
- 前端交互风格与 ChatView 一致
