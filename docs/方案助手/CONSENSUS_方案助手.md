# 方案助手 — 共识确认文档

## 需求共识

- **输入**：技术标书/需求文档（PDF/DOCX/文本粘贴）+ 可选方案模板（Word）
- **输出**：结构化的技术方案 Word 文档
- **核心引擎**：知识库 RAG 检索 + LLM 融合生成
- **定位**：与标书助手互补，标书侧重审查，方案侧重生成
- **模板驱动**：用户可上传已有方案 Word 模板，系统提取目录结构作为大纲骨架

## 技术方案共识

- 完全复用现有架构模式（Session+SQLite、SSE 流式、RAG 检索）
- 新增独立模块 `src/solution/`，不修改任何现有代码
- API 路由 `/api/solution/*`，前端新增一个 Tab 页
- 模板解析：使用 python-docx 提取 Word 文档的标题层级结构作为大纲
- 知识库检索直接调用 `api/deps.py` 的 `get_hybrid_search()`

## 边界限制

- 不涉及图表自动生成（仅文字方案）
- 不涉及多人协作编辑
- 单次方案生成不超过 20 章节
- 依赖知识库中已摄取的文档质量

## 验收标准

1. PDF/DOCX/文本三种输入方式均可正常解析
2. 大纲可自动生成并支持用户手动编辑
3. 每章内容基于 RAG 检索素材 + LLM 融合生成
4. SSE 流式输出进度，前端实时渲染
5. 导出的 Word 文档格式规范
6. 会话持久化，刷新后状态不丢失

## 所有不确定性已解决 ✅
