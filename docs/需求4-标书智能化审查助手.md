# 需求4：标书智能化审查助手

> **优先级：低**

## 一、背景

在标书定稿后，需要进行严格的内部审查：
1. 首先由标书编制者交叉审查
2. 然后由707部标书审查组分别对商务、技术等部分进行审查

由于标书编写的时间有限，为标书审查预留的时间通常情况下都很少，导致标书存在的很多问题无法及时发现，特别是**废标项**，如果有一条未响应，将导致我方无效投标，这种情况应坚决杜绝。

## 二、具体需求

本次主要完成**废标项的辅助审查**，主要流程：

1. **读取招标文件** - 通过智能体应用平台读取当前招标文件要求

2. **识别废标项条款** - 识别出招标文件中的废标项条款和要求，包括：
   - 符合性审查项
   - 资格性审查项
   - 实质性要求
   - 标"★"号项
   - 其他强制性要求

3. **生成要求清单** - 形成当前项目的招标文件实质性要求清单

4. **导入投标文件** - 导入当前项目的投标文件

5. **逐一核查** - 依据实质性要求清单，逐一查找和分析投标文件中相关响应内容：
   - 是否有缺项
   - 响应是否完整无误

6. **生成审查报告** - 最后生成废标项审查报告

## 三、核心功能点

| 功能 | 描述 |
|------|------|
| 招标文件解析 | 自动读取并解析招标文件内容 |
| 废标项识别 | 智能识别符合性审查项、资格性审查项、★号项等 |
| 要求清单生成 | 自动生成实质性要求清单 |
| 投标文件分析 | 解析投标文件，定位响应内容 |
| 响应核查 | 逐一核查是否有缺项、响应是否完整 |
| 审查报告生成 | 生成结构化的废标项审查报告 |

## 四、废标项类型

| 类型 | 说明 |
|------|------|
| 符合性审查项 | 投标文件格式、签章等基本要求 |
| 资格性审查项 | 投标人资质、资格条件等 |
| 实质性要求 | 技术参数、商务条款等核心要求 |
| ★号项 | 招标文件中标注★号的强制响应项 |
| 其他强制项 | 其他明确要求必须响应的条款 |

## 五、预期效果

- 大幅缩短标书审查时间
- 降低废标风险
- 提高审查覆盖率和准确性
- 形成标准化的审查流程和报告
- 坚决杜绝因漏项导致的无效投标
