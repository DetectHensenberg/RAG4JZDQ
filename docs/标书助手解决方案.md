# 标书助手解决方案

> 基于 Agent Zero 技能系统实现标书商务文件编写和智能化审查

## 一、需求概述

| 需求 | 优先级 | 核心功能 |
|------|--------|----------|
| 需求3：标书商务文件编写助手 | 低 | 解析招标文件、生成商务文件大纲、自动填充、证照水印 |
| 需求4：标书智能化审查助手 | 低 | 识别废标项、生成要求清单、逐项核查、生成审查报告 |

## 三、解决方案架构

### 3.1 需求3：标书商务文件编写助手

```
┌─────────────────────────────────────────────────────────────────┐
│                    标书商务文件编写助手                           │
├─────────────────────────────────────────────────────────────────┤
│  输入层                                                          │
│  ├── 招标文件 (PDF/Word)                                         │
│  └── 投标知识库 (证照、模板、声明函等)                            │
├─────────────────────────────────────────────────────────────────┤
│  处理层                                                          │
│  ├── [pdf/docx] 解析招标文件                                     │
│  ├── [literature-reader] 提取商务文件条款要求                    │
│  ├── [docx] 根据模板生成商务文件大纲                             │
│  ├── [nutrient-agent-skill] 证照添加水印                         │
│  └── [docx] 自动填充重复性内容                                   │
├─────────────────────────────────────────────────────────────────┤
│  输出层                                                          │
│  ├── 商务文件大纲 (Word)                                         │
│  ├── 带水印的证照文件 (PDF)                                      │
│  └── 填充后的承诺书/声明函 (Word)                                │
└─────────────────────────────────────────────────────────────────┘
```

#### 工作流程

1. **上传招标文件** → 使用 `pdf` 或 `docx` 技能读取文件
2. **条款识别** → 使用 `literature-reader` 提取商务文件要求条款
3. **大纲生成** → 使用 `docx` 根据既定模板生成商务文件大纲
4. **知识库调用** → 检索投标知识库中的证照、声明函等
5. **水印添加** → 使用 `nutrient-agent-skill` 为证照添加项目专用水印
6. **内容填充** → 使用 `docx` 自动填充重复性内容

#### 水印功能实现

使用 **nutrient-agent-skill** 的水印功能：

```python
# 示例：为营业执照添加水印
"给 营业执照.pdf 添加水印：仅限于 XXXX-2026-001 项目投标使用"
```

Nutrient API 支持：
- 文本水印（可自定义字体、颜色、位置、透明度）
- 图片水印
- 批量处理

### 3.2 需求4：标书智能化审查助手

```
┌─────────────────────────────────────────────────────────────────┐
│                    标书智能化审查助手                             │
├─────────────────────────────────────────────────────────────────┤
│  输入层                                                          │
│  ├── 招标文件 (PDF/Word)                                         │
│  └── 投标文件 (PDF/Word)                                         │
├─────────────────────────────────────────────────────────────────┤
│  分析层                                                          │
│  ├── [pdf/docx] 解析招标文件                                     │
│  ├── [literature-reader] 识别废标项条款                          │
│  │   ├── 符合性审查项                                            │
│  │   ├── 资格性审查项                                            │
│  │   ├── 实质性要求                                              │
│  │   └── ★号项                                                   │
│  └── 生成实质性要求清单                                          │
├─────────────────────────────────────────────────────────────────┤
│  核查层                                                          │
│  ├── [pdf/docx] 解析投标文件                                     │
│  ├── [qa-tester 逻辑] 逐项核查响应内容                           │
│  │   ├── 检查是否有缺项                                          │
│  │   └── 检查响应是否完整无误                                    │
│  └── [quality-nonconformance 逻辑] 问题分析                      │
├─────────────────────────────────────────────────────────────────┤
│  输出层                                                          │
│  ├── 实质性要求清单 (Excel/Word)                                 │
│  └── 废标项审查报告 (Word)                                       │
└─────────────────────────────────────────────────────────────────┘
```

#### 工作流程

1. **上传招标文件** → 使用 `pdf` 或 `docx` 技能读取
2. **废标项识别** → 使用 `literature-reader` 识别以下条款：
   - 符合性审查项
   - 资格性审查项
   - 实质性要求
   - 标"★"号项
3. **生成要求清单** → 使用 `xlsx` 生成结构化的实质性要求清单
4. **导入投标文件** → 使用 `pdf` 或 `docx` 解析投标文件
5. **逐项核查** → 参考 `qa-tester` 的检查机制，逐一核查：
   - 是否有缺项
   - 响应是否完整无误
6. **生成审查报告** → 使用 `docx` 生成废标项审查报告

#### 审查报告模板

```markdown
# 废标项审查报告

## 项目信息
- 项目名称：XXX
- 项目编号：XXX
- 审查日期：XXX

## 实质性要求清单
| 序号 | 条款编号 | 要求类型 | 要求内容 | 响应状态 | 响应位置 | 备注 |
|------|----------|----------|----------|----------|----------|------|
| 1    | 3.1.1    | ★号项    | ...      | ✅ 已响应 | 第X页    |      |
| 2    | 3.1.2    | 符合性   | ...      | ❌ 缺项   | -        | 需补充 |

## 审查结论
- 总计要求项：XX 项
- 已响应：XX 项
- 缺项：XX 项
- 响应不完整：XX 项

## 风险提示
1. 【高风险】第 X 项为★号项，当前缺失响应，可能导致废标
2. ...
```

## 六、自定义技能开发

### 6.1 标书条款提取技能

建议创建专门的标书条款提取技能：

```markdown
# SKILL.md

---
name: bid-document-analyzer
description: 分析招标文件，提取商务文件要求条款和废标项。当用户上传招标文件或提到"分析招标文件"、"提取废标项"时使用。
triggers:
  - 分析招标文件
  - 提取废标项
  - 商务文件要求
  - 实质性要求
---

## 功能

1. 解析招标文件（PDF/Word）
2. 识别商务文件要求条款
3. 识别废标项（符合性审查项、资格性审查项、★号项等）
4. 生成结构化的要求清单

## 识别规则

### 废标项识别关键词
- "符合性审查"
- "资格性审查"
- "实质性要求"
- "★" 或 "☆" 标记
- "否则作废标处理"
- "必须"、"应当"、"须"
- "不得"、"禁止"

### 输出格式
生成 Excel 清单，包含：
- 条款编号
- 要求类型（符合性/资格性/实质性/★号项）
- 要求内容
- 响应要求
- 备注
```

### 6.2 证照水印技能

```markdown
# SKILL.md

---
name: bid-watermark
description: 为投标证照文件添加项目专用水印。当用户需要为证照添加水印时使用。
triggers:
  - 添加水印
  - 证照水印
  - 投标水印
---

## 功能

为证照文件（营业执照、身份证、资质证书等）添加项目专用水印。

## 水印格式

"仅限于 {项目编号} 项目投标使用"

## 使用方法

1. 指定项目编号
2. 指定需要添加水印的文件
3. 调用 Nutrient API 添加水印
4. 输出带水印的文件

## 依赖

- nutrient-agent-skill
- NUTRIENT_API_KEY 环境变量
```
