# ALIGNMENT - 检索性能综合优化

## 1. 项目上下文分析
本项目是一个基于 FastAPI 的 RAG 系统，近期进行了多项核心组件的性能升级，包括 BM25 检索器的接口规范化与持久化加速、数据摄取流水线（Ingestion Pipeline）的异步化并行改造，以及多模态图像处理的预检持久化。

## 2. 需求理解确认
### 2.1 优化 2: 图像处理提速
- **目标**: 减少查询时的重复背景检测计算，提升多模态召回速度。
- **方案**: 在摄取阶段检测背景并持久化到数据库；在查询时并行执行图片评分。

### 2.2 优化 3: BM25 升级
- **目标**: 规范化稀疏检索接口，提升索引加载与保存速度。
- **方案**: 引入 `BaseSparseRetriever` 协议；使用 `pickle` 替代 `json` 进行序列化；实现原子写入。

### 2.3 优化 4: Ingestion 流水线异步化
- **目标**: 提升大规模文档摄取的吞吐量，增强用户反馈。
- **方案**: 重构 `IngestionPipeline` 为异步架构；利用 `asyncio.gather` 并行执行 LLM Transform；通过 SSE 提供实时进度。

## 3. 技术约束
- 保持与现有 `core.types` 和 `libs.splitter` 的兼容性。
- 确保异步操作对 SQLite 的 WAL 模式支持良好。
- 必须包含完整的单元测试与端到端验证。

## 4. 验收标准
- [x] BM25 索引加载速度提升 5x+。
- [x] Ingestion 流水线并发处理 LLM 任务不再阻塞。
- [x] 图像背景检测结果正确持久化。
- [x] SSE 接口正常推送进度数据。
