## 一、错误处理改进方案

### 1.1 现状分析

全项目共 162 处 `except Exception`，分布在 51 个文件中。按用途分三类：

| 类型 | 数量 | 典型位置 | 合理性 |
|------|------|----------|--------|
| **顶层兜底** | ~15 | `pipeline.py`, `knowledge_qa.py`, `protocol_handler.py` | ✅ 合理，防止未知异常导致进程崩溃 |
| **降级容错** | ~50 | `chunk_refiner.py`, `image_captioner.py`, `metadata_enricher.py` | ⚠️ 合理但应缩小范围 |
| **懒惰捕获** | ~97 | `chroma_store.py`, `document_manager.py`, `pptx_loader.py` 等 | ❌ 应细化为具体异常 |

### 1.2 改进原则

1. **顶层兜底保留** — Pipeline `run()`, MCP `protocol_handler`, Dashboard 页面入口
2. **降级容错缩小范围** — LLM/API 调用改为捕获 `(httpx.HTTPError, TimeoutError, ConnectionError, ValueError)`
3. **懒惰捕获细化** — 按实际可能的异常类型细化

### 1.3 分层细化规则

```python
# ❌ 不推荐：宽泛捕获
try:
    result = some_api_call()
except Exception as e:
    logger.error(f"Failed: {e}")

# ✅ 推荐：按异常类型分层
try:
    result = some_api_call()
except (httpx.TimeoutException, httpx.RequestError) as e:
    logger.warning(f"Network error (retryable): {e}")
    # 可重试
except (ValueError, KeyError) as e:
    logger.error(f"Data error (non-retryable): {e}")
    # 不可重试，跳过或降级
except Exception as e:
    logger.error(f"Unexpected error: {e}", exc_info=True)
    # 仅在顶层兜底
```

### 1.4 核心模块细化清单

#### A. LLM/Embedding 层（httpx 网络调用）

| 文件 | 当前 | 改为 |
|------|------|------|
| `openai_llm.py` | `except Exception` | `except (httpx.TimeoutException, httpx.RequestError, OpenAILLMError)` |
| `openai_embedding.py` | `except Exception` | `except (openai.APIError, openai.APITimeoutError, OpenAIEmbeddingError)` |
| `openai_vision_llm.py` | `except Exception` | `except (httpx.TimeoutException, httpx.RequestError)` |

#### B. 存储层（SQLite + 文件 I/O）

| 文件 | 当前 | 改为 |
|------|------|------|
| `chroma_store.py` | `except Exception` | `except (RuntimeError, chromadb.errors.ChromaError)` |
| `image_storage.py` | `except Exception` | `except (sqlite3.Error, IOError, FileNotFoundError)` |
| `bm25_indexer.py` | `except Exception` | `except (json.JSONDecodeError, IOError, ValueError)` |
| `file_integrity.py` | `except Exception` | `except (sqlite3.Error, IOError)` |

#### C. 文档加载层

| 文件 | 当前 | 改为 |
|------|------|------|
| `pdf_loader.py` | `except Exception` | `except (IOError, ValueError, RuntimeError)` |
| `pptx_loader.py` | `except Exception` | `except (IOError, ValueError, KeyError, zipfile.BadZipFile)` |

#### D. 保留顶层兜底（不改）

## 二、代码重复改进方案

### 2.1 现状分析

`pipeline.py` 的 `run()` 方法中，5 个 `trace.record_stage()` 调用各自手动组装大段字典数据，模式高度相似：

```python
if trace is not None:
    trace.record_stage("load", {
        "method": "markitdown",
        "doc_id": document.id,
        "text_length": len(document.text),
        ...
    }, elapsed_ms=_elapsed)
```

**问题**:
- 每个 Stage 都有 10-30 行数据组装代码
- `if trace is not None:` 重复 5 次
- chunk 详情列表推导式几乎相同

### 2.2 改进方案: 提取 Trace 辅助方法

```python
class IngestionPipeline:
    
    def _trace_stage(
        self, 
        trace: Optional[TraceContext], 
        stage_name: str, 
        data: Dict[str, Any], 
        elapsed_ms: float = 0.0
    ) -> None:
        """Record a pipeline stage in the trace context (if enabled)."""
        if trace is not None:
            trace.record_stage(stage_name, data, elapsed_ms=elapsed_ms)
    
    @staticmethod
    def _chunk_summary(chunks: List[Chunk]) -> List[Dict[str, Any]]:
        """Build a serializable chunk summary for trace recording."""
        return [
            {
                "chunk_id": c.id,
                "text": c.text,
                "char_len": len(c.text),
                "chunk_index": c.metadata.get("chunk_index", i),
            }
            for i, c in enumerate(chunks)
        ]
    
    @staticmethod
    def _chunk_transform_summary(
        chunks: List[Chunk], 
        pre_refine_texts: Dict[str, str]
    ) -> List[Dict[str, Any]]:
        """Build transform stage chunk details for trace."""
        return [
            {
                "chunk_id": c.id,
                "text_before": pre_refine_texts.get(c.id, ""),
                "text_after": c.text,
                "char_len": len(c.text),
                "refined_by": c.metadata.get("refined_by", ""),
                "enriched_by": c.metadata.get("enriched_by", ""),
                "title": c.metadata.get("title", ""),
                "tags": c.metadata.get("tags", []),
                "summary": c.metadata.get("summary", ""),
            }
            for c in chunks
        ]
```

**优化效果**:
- `run()` 中每个 trace 块从 10-30 行缩减到 3-5 行
- 数据格式集中管理，改一处全生效
- `if trace is not None` 检查收拢到 `_trace_stage()` 一处

### 2.3 实施策略

1. 在 `IngestionPipeline` 类中添加 `_trace_stage()`, `_chunk_summary()`, `_chunk_transform_summary()` 辅助方法
2. 逐步替换 `run()` 中 5 个 `trace.record_stage()` 调用
3. 确保测试通过（`test_pipeline_progress.py`）

## 五、不改动项（设计决策）

| 项目 | 原因 |
|------|------|
| Pipeline 顶层 `except Exception` | 必须兜底，防止单文件失败影响批量处理 |

| Transform 降级 `except Exception` | LLM 调用可能抛出任意异常，降级到 rule-based 是正确策略 |
