# 2026-03-12 摄取管道修复与 ChromaDB 索引修复

## 概述
本次会话主要解决了 Ingestion Pipeline 的多个问题，包括文件加载器参数不匹配、embedding batch size 过大、文本超长、图片格式不支持、以及 ChromaDB HNSW 索引损坏等关键问题。

---

## 执行操作记录

### 1. PdfLoader 参数不匹配修复
**问题**: `LoaderFactory.create()` 传递 `vision_llm` 参数给 `PdfLoader`，但 `PdfLoader.__init__()` 不接受该参数
**错误信息**: `TypeError: PdfLoader.__init__() got an unexpected keyword argument 'vision_llm'`
**解决方案**: 在 `PdfLoader.__init__()` 中添加 `vision_llm: Any = None` 参数（兼容但不使用）
**修改文件**: `src/libs/loader/pdf_loader.py:58-62`

### 2. Embedding Batch Size 对齐修复
**问题**: `BatchProcessor` 使用 `batch_size=100`，而 `DenseEncoder` 使用 `batch_size=6`，导致两层分批不对齐，单个 API 失败会丢失 100 个 chunk
**错误信息**: `Encoding failed: got 200 vectors for 248 chunks`
**解决方案**: 将 `BatchProcessor` 的 `batch_size` 也改为 `embedding_batch_size`（6），与 `DenseEncoder` 对齐
**修改文件**: `src/ingestion/pipeline.py:175-180`

### 3. 文本超长截断修复
**问题**: DashScope `text-embedding-v3` 模型限制 8192 token，部分 chunk 文本超长导致 embedding 失败
**错误信息**: `Range of input length should be [1, 8192]`
**解决方案**: 在 `OpenAIEmbedding.embed()` 中添加文本截断，限制最大 6000 字符（约 4000 token 安全边界）
**修改文件**: `src/libs/embedding/openai_embedding.py:146-149`

### 4. Pipeline 容错处理
**问题**: 少量 chunk embedding 失败会导致整个 pipeline 中断
**解决方案**: 修改向量存储前的检查逻辑，允许部分失败，跳过失败的 chunk 继续存储成功的部分
**修改文件**: `src/ingestion/pipeline.py:440-454`

### 5. 不支持图片格式过滤
**问题**: PPTX 中的 `.x-wmf` 格式图片不被 Vision API 支持，导致 captioning 失败
**错误信息**: `The image format is illegal and cannot be opened`
**解决方案**: 在 `ImageCaptioner._get_caption()` 中添加图片格式检查，跳过 WMF/EMF 等不支持的格式
**修改文件**: `src/ingestion/transform/image_captioner.py:118-123`

### 6. markitdown[docx] 依赖安装
**问题**: DOCX 文件解析失败，缺少 `mammoth` 依赖
**错误信息**: `DocxConverter threw MissingDependencyException`
**解决方案**: 安装 `pip install markitdown[docx]`
**命令**: `pip install "markitdown[docx]"`

### 7. ChromaDB HNSW 索引损坏修复
**问题**: ChromaDB 的 FTS5 全文搜索索引损坏，导致无法初始化
**错误信息**: `Error loading hnsw index` / `malformed inverted index for FTS5 table`
**根因分析**:
- 进程强制终止时 SQLite 写入中断
- Windows 文件锁机制不如 Unix 稳定
- 频繁开关数据库连接
**解决方案**:
1. 启用 SQLite WAL 模式提高稳定性
2. 删除损坏的 FTS5 表，保留核心数据（3343 embeddings, 51095 metadata）
3. ChromaDB 启动时自动重建 FTS5 索引
**修改文件**: `src/libs/vector_store/chroma_store.py:110-131`
**修复脚本**: `remove_fts.py`
